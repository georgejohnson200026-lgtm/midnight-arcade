# Midnight Arcade

Static arcade site with multiple browser games, ready for GitHub Pages deployment.

## GitHub Pages setup

1. Create a GitHub repository and push this project to the `main` branch.
2. In GitHub, open `Settings` -> `Pages`.
3. Under `Build and deployment`, set `Source` to `GitHub Actions`.
4. The workflow in `.github/workflows/deploy-pages.yml` will publish the site automatically on every push to `main`.

## Custom domain

This project already includes `CNAME` with `midnightarcade.com`.

To make the custom domain work:

1. Own the domain `midnightarcade.com`.
2. In your domain registrar, point the domain to GitHub Pages.
3. For the apex domain, create these A records:
   - `185.199.108.153`
   - `185.199.109.153`
   - `185.199.110.153`
   - `185.199.111.153`
4. For `www`, create a CNAME record pointing to `<your-github-username>.github.io`.
5. Back in GitHub Pages settings, confirm `midnightarcade.com` is shown as the custom domain.
6. Enable `Enforce HTTPS` after DNS finishes propagating.

## Local preview

Open the site locally with your existing static server or any simple web server.

## Real online multiplayer setup

The multiplayer invite/chat system uses Firebase Realtime Database so links can work between different devices/users.

1. Create a Firebase project at https://console.firebase.google.com.
2. In the project, create a Realtime Database (start in test mode for initial setup).
3. In `Project settings` -> `General` -> `Your apps`, create a Web app and copy the web config values.
4. Update `firebase-config.js` with your real Firebase config values.
5. In Realtime Database `Rules`, set temporary rules for testing:

```json
{
   "rules": {
      ".read": true,
      ".write": true
   }
}
```

6. Deploy the site to GitHub Pages (steps above).
7. Share the live `hangman.html` or `tic-tac-toe.html` link with `?room=ROOMID` (generated by Create Invite Link).

### Important

- `firebase-config.js` must contain your real values before multiplayer works online.
- The current Firebase rules above are open for quick testing only. Tighten rules before production.
